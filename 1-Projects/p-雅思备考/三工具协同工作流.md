# 三工具协同工作流

## 日常循环

### 🌅 早上：飞书 + Claude

1. **飞书** 推送今日学习计划 + 任务提醒
2. 开始学习某个模块（如听力 Section 3）
3. 遇到问题 → **Claude** 即时答疑
4. Claude 解释完 → "**保存到 Obsidian**" → 知识点入库

### 🌤️ 学习中间

1. 做完一套题 → 对答案
2. 错题发给 **Claude**："分析这几道错题的原因"
3. Claude 给出分析 + 改进建议
4. "**保存到 Obsidian 错题本**"
5. **飞书** 记录今日完成的任务 + 时长

### 🌙 晚上：三者汇总

1. **飞书** 弹出打卡提醒
2. 在 **Obsidian** 写当日学习日志（或直接说给 Claude）
3. **Claude** 生成学习周报 → 手动贴到飞书文档
4. 更新飞书多维表格的完成状态

---

## 每周循环

### 周末：Claude 驱动的复盘

对 Claude 说：
> "帮我回顾这周 Obsidian 的学习日志，总结：
> 1. 各模块投入时间占比
> 2. 错题集中在哪些题型
> 3. 词汇量增长趋势
> 4. 下周重点建议"

Claude 会读取 Obsidian 中本周所有日志和错题记录，生成分析报告。

### 模考周

1. 飞书日历安排模考时间
2. 模考后 → 成绩填入飞书表格
3. 各模块错题 → Claude 逐题分析
4. 分析结果 → Obsidian 归档
5. Claude 根据趋势调整下周计划 → 飞书任务更新

## 具体场景示例

### 场景 1：阅读错题分析

```
你说：这篇文章第 7、12、23 题我错了，帮我看看为什么

Claude 会：
1. 逐题分析错误类型（定位、同义替换、推断、词汇…）
2. 指出每种错误根因
3. 给出避免同类错误的策略
4. "保存到 Obsidian" → 按题型归类到错题本
```

### 场景 2：作文批改

```
你说：这是我写的 Task 2 作文，帮我按雅思标准评分批改

Claude 会：
1. 四个维度评分（Task Response, Coherence, Lexical, Grammar）
2. 逐句批注问题
3. 给出 7 分版本改写
4. "保存到 Obsidian" → 归档到作文进化，追踪进步
```

### 场景 3：口语模拟

```
你说：模拟雅思口语 Part 2，"描述一个你喜欢的建筑"

Claude 会：
1. 扮演考官，计时 1 分钟准备 + 2 分钟叙述
2. 追问 Part 3 的问题
3. 给出评分 + 改进建议
4. "保存到 Obsidian" → 语料库扩充
```
