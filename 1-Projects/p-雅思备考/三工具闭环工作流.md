# 三工具闭环工作流

> **核心变化（2026-06-17）：Claude 现已通过 MCP 直连飞书 API，不再是旁观者，而是三工具闭环的中央枢纽。**

---

## 架构总览

```
                    ┌─────────────────────────┐
                    │                         │
                    │      Claude (中枢)       │
                    │   思考·分析·调度·转化     │
                    │                         │
                    └───┬──────────────┬──────┘
                        │              │
           ┌────────────▼──┐      ┌───▼────────────┐
           │   飞书 MCP     │      │  Obsidian MCP   │
           │               │      │                 │
           │ 📋 任务 CRUD   │      │ 📝 笔记 CRUD     │
           │ 📄 文档 CRUD   │      │ 🔍 全文搜索      │
           │ 📚 知识库管理  │      │ 🏷️ 标签/图谱    │
           │ 📁 云盘操作    │      │ 📅 日记系统      │
           │ 🔔 提醒通知    │      │ 🔗 双向链接      │
           └───────────────┘      └─────────────────┘
             执行层 (管)              知识层 (存)
```

**三个工具，两种 MCP 通道，Claude 一手调度。**

## 数据流动方向

### 飞书 → Obsidian（知识沉淀）

| 飞书来源 | Claude 处理 | Obsidian 落点 |
|----------|------------|---------------|
| 已完成任务 | 整理学习成果 | `学习日志.md` |
| 模考多维表格 | 趋势分析 + 薄弱诊断 | `模考记录.md` |
| 飞书文档（PDF/资料） | 提取关键知识点 | `3-Resources/` 对应 MOC |
| 飞书提醒/打卡记录 | 汇总学习时长 | `学习日志.md` 周报 |
| 知识库文档 | 结构化重组 | 对应知识节点 |

### Obsidian → 飞书（行动转化）

| Obsidian 来源 | Claude 处理 | 飞书落点 |
|--------------|------------|---------|
| 错题本薄弱项 | 生成针对性任务 | 飞书任务（带截止日期）|
| 学习日志趋势 | 制定下周计划 | 飞书任务清单 |
| 模考分析报告 | 提取行动项 | 飞书表格 + 提醒 |
| 词汇掌握缺口 | 生成 Anki/练习 | 飞书文档 |
| 作文进化记录 | 推荐重点训练方向 | 飞书任务 |

## 闭环工作流场景

### 🔄 每日闭环

```
08:00  Claude 读取飞书今日任务
       ↓
       学习过程中 → Claude 答疑、批改、模拟
       ↓
22:00  Claude 将学习成果写入 Obsidian
       + Claude 更新飞书任务完成状态
       + Claude 创建明日任务到飞书
```

**Claude 要说的话：**
> "今天完成了听力 Section 2 练习，做错 3 道地图题。已保存到错题本。明天建议重点攻克地图题，任务已创建到飞书。"

### 🔄 模考闭环

```
你：飞书里有最新的模考成绩，帮我分析

Claude：
  1. 读取飞书多维表格的模考数据
  2. 分析各模块趋势（上升/下降/平台期）
  3. 找出高频错误题型
  4. 生成分析报告 → 写入 Obsidian
  5. 基于分析 → 在飞书创建针对性任务
  6. 可设置飞书提醒（下次模考日期）
```

### 🔄 复习闭环

```
Claude 定期扫描 Obsidian 错题本 → 发现某题型超过 7 天未复习
       ↓
       生成针对性练习
       ↓
       在飞书创建"复习听力地图题"任务 + 设置提醒
       ↓
       练习后 → 错题本更新掌握状态
```

### 🔄 周报闭环

```
每周日：
  Claude 读取本周 Obsidian 所有学习日志
  + 读取飞书任务完成情况
  ↓
  生成周报：
    - 各模块投入时间
    - 错题分布热力图
    - 进步/退步趋势
    - 下周重点建议
  ↓
  周报同时写入：
    → Obsidian（永久存档，可回溯）
    → 飞书文档（可分享、可协作）
```

## CLI 命令速查

在 Claude Code 中可以这样说：

| 你说 | 效果 |
|------|------|
| "帮我查看飞书待办" | 列出所有任务 |
| "创建飞书任务：做 TG 听力 Section 3" | 在飞书创建任务 |
| "把今天的错题分析保存到 Obsidian" | 写入错题本 |
| "分析飞书模考表格的趋势" | 读飞书数据 → 分析 → 写 Obsidian |
| "根据 Obsidian 错题本生成本周飞书任务" | 反向闭环 |
| "生成本周学习周报，同时存 Obsidian 和飞书" | 双写闭环 |
| "搜索 Obsidian 里关于地图题的笔记" | 知识检索 |

## 与旧版工作流的区别

| | 旧版（2026-06-16） | 新版（2026-06-17+） |
|------|------|------|
| 飞书 → Claude | ❌ 手动复制粘贴 | ✅ MCP 直读 |
| Claude → 飞书 | ❌ 手动粘贴 | ✅ MCP 直写 |
| 闭环能力 | ❌ 开环 | ✅ 完全闭环 |
| Claude 角色 | 旁观导师 | 中央调度中枢 |
| 自动化程度 | 低（手动搬运） | 高（一句话触发全链路） |
