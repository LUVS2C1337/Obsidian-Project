# 🛠️ 敏捷工具与技术

---

## 估算技术

| 技术 | 说明 | 适用场景 |
|------|------|---------|
| **规划扑克** (Planning Poker) | 团队同时出牌，讨论差异较大的估算 | 故事点估算 |
| **类推估算** (Analogous) | 参照历史项目类似功能 | 早期估算 |
| **三点估算** (Three-Point) | 乐观+悲观+最可能 / 3 | 更精确的单任务估算 |
| **亲和估算** (Affinity Estimating) | 快速对大量用户故事分组排序 | Product Backlog 梳理 |
| **T 恤尺寸** (T-Shirt Sizing) | XS/S/M/L/XL 粗略分类 | 高层级规划 |

---

## 优先级排序技术

| 技术 | 说明 |
|------|------|
| **MoSCoW** | Must / Should / Could / Won't |
| **Kano 模型** | 基本型→期望型→兴奋型需求分类 |
| **价值 vs 风险矩阵** | 按价值和风险两个维度排列优先级 |
| **加权最短作业优先 (WSJF)** | Cost of Delay / Job Duration，SAFe 框架常用 |

## 用户故事 (User Story)

**标准格式：**
> **As a** [用户角色]
> **I want** [功能需求]
> **So that** [业务价值]

**验收标准 (Acceptance Criteria) — INVEST 原则：**
- **I**ndependent — 独立
- **N**egotiable — 可协商
- **V**aluable — 有价值
- **E**stimable — 可估算
- **S**mall — 足够小
- **T**estable — 可测试

## 考试高频场景

| 场景 | 正确做法 |
|------|---------|
| 团队估算差异大 | 让意见不同双方各自解释理由，不要取平均 |
| 变更需求涌入 | PO 重新排优先级，不直接中断 Sprint |
| Sprint 目标有风险 | 调整范围，不延长 Sprint |
| 团队冲突 | 回顾会议上解决，Scrum Master 引导 |
| 客户不清楚需求 | 用原型 (Prototype) 验证，短迭代快速试错 |
