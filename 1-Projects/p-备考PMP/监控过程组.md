# 📊 监控过程组

> **目的：** 跟踪、审查和调整项目进展，识别变更并触发变更控制。
> **核心问题：** 实际 vs 计划的偏差有多少？需要纠偏吗？

## ⭐ 超级重点：4.6 实施整体变更控制

| 要素 | 说明 |
|------|------|
| **提出变更** | 任何相关方均可提出 |
| **CCB审批** | 变更控制委员会审批，项目经理是成员但不是决策者 |
| **影响** | 已批准的变更影响所有基准/计划 |
| **例外** | 紧急情况下可先实施后补审批 |

### 变更流程
```
识别变更 → 记录问题日志 → 评估影响 → 提交CCB → 
批准/拒绝 → 更新项目管理计划 → 通知相关方 → 执行
```

## 🔑 两大引擎

### 确认范围 (5.5) vs 控制质量 (8.3)

| | 确认范围 | 控制质量 |
|--|---------|---------|
| **对象** | 验收可交付成果 | 核实的可交付成果 |
| **焦点** | 客户/发起人正式验收 | 检测是否满足质量要求 |
| **顺序** | 先控制质量 → 再确认范围 |

### 挣值管理 (EVM)

| 指标 | 公式 | 含义 |
|------|------|------|
| **PV** | 计划价值 | 截至某时间点应完成的价值 |
| **EV** | 挣值 | 实际完成了多少 |
| **AC** | 实际成本 | 花了多少钱 |
| **SV** | EV - PV | >0 进度超前 |
| **CV** | EV - AC | >0 成本节约 |
| **SPI** | EV/PV | >1 进度超前 |
| **CPI** | EV/AC | >1 成本节约 |
| **EAC** | BAC/CPI | 完工估算 |
| **TCPI** | (BAC-EV)/(BAC-AC) | 完工尚需绩效指数 |

> 考试一定要会算！TCPI 常用 BAC 做分母，但超过预算时用 EAC。
