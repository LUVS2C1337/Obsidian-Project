# 📋 治理与监控

## CloudWatch

| 组件 | 用途 |
|------|------|
| **指标 (Metrics)** | CPU · 内存 · 磁盘 · 网络 — 标准/自定义 |
| **告警 (Alarms)** | 指标触发 → SNS/ASG/Actions |
| **日志 (Logs)** | 收集应用日志 · 存储/查询/告警 |
| **仪表盘 (Dashboards)** | 可视化监控 |
| **Synthetics** | Canary — 模拟用户请求 (脚本) |
| **Agent** | 在 EC2/本地服务器上收集系统级指标 |

### 关键概念
- **标准指标** — 每 5 min (基础) / 每 1 min (详细监控，额外收费)
- **CloudWatch 代理** — 收集内存/磁盘/日志 → CloudWatch Logs
- **日志洞察 (Logs Insights)** — SQL 风格查询日志

## CloudTrail（审计）

| 概念 | 要点 |
|------|------|
| **默认启用** | 管理事件 (控制台/SDK/CLI 操作) |
| **数据事件** | S3 GetObject · Lambda 调用 (需额外开启) |
| **多区域** | 开启多区域 Trail 统一审计 |
| **组织** | 组织级 Trail — 所有账户审计 |
| **存储** | S3 + CloudWatch Logs (可选) |

> **CloudWatch 监控性能 · CloudTrail 审计操作** — 这是最基础的区别

## Config（合规评估）

| 概念 | 要点 |
|------|------|
| **功能** | 资源配置变更记录 + 合规评估规则 |
| **规则** | AWS 托管规则 + 自定义规则 (Lambda) |
| **时间线** | 查看资源的历史配置 |
| **修正** | 自动修正不合规资源 (SSM Automation) |

## AWS Organizations

| 概念 | 要点 |
|------|------|
| **管理账户** | Master/Payer 账户 |
| **OU** | 组织单元 — 层级结构 |
| **SCP** | 服务控制策略 — OU/账户级别权限边界 |
| **整合账单** | 统一付费，预留实例/节省计划共享 |
| **Account Factory** | 自动创建标准化账户 |

## AWS Systems Manager (SSM)

| 概念 | 要点 |
|------|------|
| **Session Manager** | 无 SSH/RDP 访问实例 (安全) |
| **Patch Manager** | 自动打补丁 |
| **Automation** | 自动化运维任务 |
| **Parameter Store** | 参数存储 (加密/明文) |
| **Run Command** | 批量执行命令 |
| **Inventory** | 收集实例配置信息 |
