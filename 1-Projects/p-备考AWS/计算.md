# 🖥️ 计算

## EC2（弹性计算云）

| 概念 | 要点 |
|------|------|
| **实例类型** | 通用 (t/m) · 计算优化 (c) · 内存优化 (r/x) · 存储优化 (i/d/h) · GPU (p/g) |
| **购买选项** | 按需 · 预留 (RI: 1/3年, 标准/可转换) · Spot · 专用主机 · 专用实例 |
| **存储选项** | EBS (持久块存储) · 实例存储 (临时) · EFS (共享文件系统) |
| **安全** | Security Groups (SG) — 实例级防火墙 · NACL — 子网级防火墙 |
| **用户数据** | 启动时执行的脚本 (cloud-init) |

### 关键考点
- **Spot 实例** — 便宜 90%，可中断，适合容错/无状态/批处理
- **预留实例** — 标准不可改实例类型，可转换可以改
- **专用主机** — 自带许可 (BYOL)，硬件隔离
- **Placement Group** — Cluster (低延迟) / Spread (高可用) / Partition (大数据)

## Lambda（无服务器计算）

| 概念 | 要点 |
|------|------|
| **最大运行时间** | 15 分钟 |
| **支持的触发器** | S3, API Gateway, DynamoDB Streams, SQS, EventBridge |
| **并发限制** | 账户级 Soft Limit 1000 (可提额) |
| **冷启动** | VPC 内的 Lambda 冷启动更慢 (需 ENI) |
| **版本/别名** | 不可变版本 · 别名实现蓝绿部署 |

### 关键考点
- Lambda@Edge — 在 CloudFront 边缘执行
- 适合短运行、事件驱动的任务，不适合长时间运行
- **预置并发 (Provisioned Concurrency)** — 解决冷启动

## Auto Scaling

| 概念 | 要点 |
|------|------|
| **启动模板** | AMI + 实例类型 + SG + 用户数据 |
| **伸缩策略** | 动态 (基于指标) / 计划 (按时间) / 预测 (ML) |
| **健康检查** | EC2 状态检查 / ELB 健康检查 |
| **冷却时间** | 伸缩后等待再伸缩，防振荡 |

## Elastic Load Balancer (ELB)

| 类型 | 层级 | 特点 |
|------|------|------|
| **ALB** | L7 | HTTP/HTTPS · 基于路径/主机名路由 · WebSocket |
| **NLB** | L4 | TCP/UDP · 超低延迟 · 静态 IP (EIP) |
| **CLB** | L4/7 | 经典，不建议新项目用 |

**SAA 高频场景：** ELB + ASG → 高可用 Web 架构
**SAP 高频场景：** 跨区域 ASG + Route 53 加权路由 + 蓝绿部署
