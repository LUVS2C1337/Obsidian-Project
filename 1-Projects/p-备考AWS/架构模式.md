# 🏗️ 架构设计模式

> SAA 考"用哪个服务"，SAP 考"这个架构怎么设计"。

---

## 一、高可用架构模式

### 1. 多 AZ 部署 (Multi-AZ)

```
Route 53 (故障切换路由)
    │
    ├── ALB ─────────────── ALB
    │                       │
    ├── us-east-1a          ├── us-east-1b
    │   EC2 + ASG           │   EC2 + ASG
    │   RDS Multi-AZ        │   RDS 主/备
    │   ElastiCache 备      │
```

### 2. 多区域部署 (Multi-Region)

```
Route 53 (延迟/故障切换)
    ├── us-east-1          ├── eu-west-1
    │   Full Stack         │   Full Stack
    │   Aurora Global      │   Aurora 备
    │   S3 CRR             │   S3 复制
```

**关键：** 被动备区 vs 主动-主动

### 3. 无服务器高可用

```
CloudFront (全球边缘)
    └── API Gateway
         └── Lambda (多 AZ)
              ├── DynamoDB (多区域自动复制)
              ├── S3 (11个9持久性)
              └── SQS (多 AZ 存储)
```

## 二、容灾模式

| 模式 | RTO | RPO | 成本 | 适合场景 |
|------|-----|-----|------|---------|
| **备份+还原** | 小时级 | 24h | $ | 非关键系统 |
| **Pilot Light** | 10-60min | 几分钟 | $$ | 中等关键 |
| **Warm Standby** | ~5min | 秒级 | $$$ | 关键系统 |
| **Multi-site** | ~1min | 近零 | $$$$ | 核心系统 |

## 三、缓存模式

```
用户请求
    │
    ├── CloudFront (边缘缓存)
    │
    ├── ElastiCache (Redis/Memcached)
    │    └── RDS (后端数据库)
    │
    ├── DAX (DynamoDB 加速)
    │    └── DynamoDB
    │
    ├── S3 Transfer Acceleration
         └── S3
```

### 适用场景
| 缓存 | 缓存什么 | 场景 |
|------|---------|------|
| CloudFront | 静态文件 · 图片 · API 响应 | Web 加速 |
| ElastiCache | 数据库查询 · 会话 · 页面 | 减轻 RDS 压力 |
| DAX | DynamoDB 查询 | 微秒级响应 |
| S3 TA | 上传加速 | 全球用户上传 |

## 五、微服务与事件驱动

```
用户 → API Gateway
         ├── Lambda: 订单服务 → DynamoDB
         │     └── SNS ─── SQS ─── Lambda: 通知
         ├── Lambda: 支付服务 → DynamoDB
         └── Step Functions 编排
               ├── 检查库存 → 扣款 → 通知 → ...
               └── DLQ (失败处理)
```

## SAP 高频面试题式考点

| 问题 | 方案 |
|------|------|
| 迁移本地应用到云 | 7R 策略 → 通常先 Rehost 再 Refactor |
| 全球用户低延迟 | CloudFront + 多区域部署 + Route 53 延迟路由 |
| 数据库读写分离 | RDS 只读副本 (读) + Multi-AZ (写高可用) |
| 安全合规审计 | CloudTrail + Config + GuardDuty + Security Hub |
| 混合云网络 | Direct Connect + VPN 备份 |
| 跨账户共享资源 | RAM (Resource Access Manager) |
| 容器编排 | ECS Fargate (无服务器) / EKS (K8s) |
| 蓝绿部署 | Route 53 加权 · CodeDeploy · Lambda 别名 |
