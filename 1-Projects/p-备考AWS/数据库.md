# 🗄️ 数据库

## RDS（关系型数据库服务）

| 概念 | 要点 |
|------|------|
| **引擎** | MySQL · PostgreSQL · MariaDB · Oracle · SQL Server |
| **多可用区** | 同步复制到备区，自动故障切换 (DR) |
| **只读副本** | 最多 15 个 · 异步复制 · 可跨区域 |
| **备份** | 自动备份 (保留 35 天) · 手动快照 (永久) |
| **存储** | gp2/gp3 · io1/io2 (最高 256K IOPS) |

### 关键考点
- **多 AZ 不是扩展，是容灾** — 故障自动切换到备区
- **只读副本是扩展读能力** — 应用需修改连接
- **蓝绿部署** — 用 RDS Blue/Green Deployment 零停机变更数据库

## Aurora（AWS 自研数据库）

| 概念 | 要点 |
|------|------|
| **兼容** | MySQL · PostgreSQL |
| **性能** | MySQL 5x · PostgreSQL 3x 标准 RDS |
| **存储** | 自动扩展 (10GB ~ 128TB)，按使用付费 |
| **高可用** | 6 副本跨 3 AZ，自动故障切换 < 30s |
| **Global** | 跨区域复制（1s 以内延迟）|

### Aurora vs RDS
| | RDS | Aurora |
|--|-----|--------|
| 性能 | 标准 | 5x MySQL / 3x PG |
| 存储 | 手动配置 | 自动扩展 |
| 故障切换 | 几十秒到分钟 | < 30s |
| 成本 | 固定 | 按需 |

## DynamoDB（NoSQL 键值/文档数据库）

| 概念 | 要点 |
|------|------|
| **模型** | 键值 + 文档 (JSON) |
| **性能** | 个位数毫秒，DAX 加速到微秒 |
| **扩展** | 自动扩容 (水平) |
| **一致性** | 最终一致性 (默认) / 强一致性 |
| **DAX** | DynamoDB Accelerator — 内存缓存，微秒级响应 |

### 关键考点
- **RCU / WCU** — 读写容量单位，按需/预留模式
- **TTL** — 自动过期删除，降成本
- **流 (Streams)** — Lambda 触发、跨区域复制
- **DynamoDB Accelerator (DAX)** — 读密集型场景的缓存层
- **全局表 (Global Tables)** — 多区域主主复制

## ElastiCache（内存缓存）

| 引擎 | 特点 | 场景 |
|------|------|------|
| **Redis** | 丰富数据结构 · 持久化 · 发布/订阅 · 排序集 | 缓存 · 会话管理 · 排行榜 |
| **Memcached** | 简单键值 · 分片 · 无持久化 | 简单缓存 · CPU 密集 |

**典型架构：** RDS + ElastiCache = 减轻数据库读压力

---

**SAA 高频选择题：** 读密集型 → ElastiCache / DynamoDB DAX；写密集型 → DynamoDB；复杂查询 → RDS/Aurora
