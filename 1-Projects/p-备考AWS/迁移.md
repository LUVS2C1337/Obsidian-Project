# 🚚 迁移

## 迁移策略 — 7R

| 策略 | 说明 | 场景 |
|------|------|------|
| **Rehost** (Lift & Shift) | 直接迁移至 EC2 | 最快迁移 |
| **Replatform** (Lift & Reshape) | 小改动利用托管服务 (RDS) | 性能/成本优化 |
| **Refactor/Re-architect** | 重构为云原生 (微服务/Serverless) | 长期收益最大 |
| **Repurchase** | 换产品 (如 CRM 换 Salesforce) | 功能升级 |
| **Retire** | 关停不再需要的应用 | 降成本 |
| **Retain** | 保留在本地 | 合规/依赖 |

## 迁移服务

| 服务 | 用途 |
|------|------|
| **AWS Migration Hub** | 集中追踪迁移进度 |
| **DMS (Database Migration Service)** | 数据库迁移 (同构/异构) + 持续复制 |
| **SMS (Server Migration Service)** | 服务器迁移 (VMware/Hyper-V → EC2) |
| **DataSync** | 大规模数据从本地 → S3/EFS |
| **Snowball / Snowmobile** | 物理设备传输海量数据 (PB/EB 级) |
| **Application Discovery Service** | 发现本地应用依赖关系 |
| **Migration Evaluator** | TCO (总拥有成本) 分析 |

## Storage Gateway（混合存储）

| 类型 | 用途 |
|------|------|
| **File Gateway** | SMB/NFS → S3 (本地缓存低频访问) |
| **Volume Gateway (Cached)** | iSCSI → S3 (缓存热点) |
| **Volume Gateway (Stored)** | iSCSI → 本地全量 + 异步备份 S3 |
| **Tape Gateway** | 虚拟磁带库 → S3/Glacier |
