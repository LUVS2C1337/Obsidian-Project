# 💾 存储

## S3（简单存储服务）

| 概念 | 要点 |
|------|------|
| **持久性** | 99.999999999% (11个9) |
| **可用性** | 标准 99.99% |
| **最小对象** | 0 字节 |
| **最大对象** | 5 TB（单次 PUT 最大 5 GB，大文件用 Multipart Upload） |

### 存储类

| 存储类 | 可用性 | 检索时间 | 场景 |
|--------|--------|---------|------|
| **S3 Standard** | 99.99% | 即时 | 频繁访问 |
| **S3 Intelligent-Tiering** | 99.9% | 即时 | 访问模式不确定 |
| **S3 Standard-IA** | 99.9% | 即时 | 低频访问 |
| **S3 One Zone-IA** | 99.5% | 即时 | 可重建数据 |
| **S3 Glacier Instant Retrieval** | 99.9% | 毫秒 | 归档需秒级访问 |
| **S3 Glacier Flexible** | 99.99% | 1-5 min (加急) | 归档备查 |
| **S3 Glacier Deep Archive** | 99.99% | 12h | 长期归档 |

### 关键功能
- **版本控制** — 保留所有版本，防误删
- **生命周期策略** — 自动转换存储类 → 过期删除
- **预签名 URL** — 临时授权访问私有对象
- **静态网站托管** — S3 直接托管静态网站
- **跨区域复制 (CRR)** — 自动跨区域复制对象
- **S3 对象锁定** — WORM 合规 (Write Once Read Many)

### S3 加密
- SSE-S3 (AES-256) — AWS 管理密钥
- SSE-KMS — KMS 管理密钥
- SSE-C — 客户提供密钥
- 客户端加密 — 上传前自行加密

## EBS（弹性块存储）

| 概念 | 要点 |
|------|------|
| **类型** | gp2/gp3 (通用) · io1/io2 (高性能 SSD) · st1 (吞吐 HDD) · sc1 (冷 HDD) |
| **特点** | 同一 AZ · 可挂载/卸载 · 快照备份到 S3 |
| **加密** | EBS 加密默认可启用，用 KMS |
| **快照** | 增量备份 · 可跨区域复制 · 从快照建新卷 |

### EBS vs 实例存储
| | EBS | 实例存储 |
|--|-----|---------|
| 持久性 | ✅ 实例终止后保留 | ❌ 实例终止即丢失 |
| 速度 | 取决于类型 | 更高 IOPS |
| 备份 | 快照 | 不能快照 |

## EFS（弹性文件系统）

| 概念 | 要点 |
|------|------|
| **协议** | NFSv4 |
| **扩展** | 自动伸缩，按使用付费 |
| **性能** | 通用 (General Purpose) / Max I/O |
| **存储类** | Standard / Standard-IA / One Zone |
| **访问** | SG + EFS 策略 + POSIX 权限 |

> **关键区分：** S3 = 对象存储 · EBS = 块存储 (单实例) · EFS = 文件存储 (多实例共享)
