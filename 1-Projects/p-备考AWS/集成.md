# 🔗 集成（消息与 API）

## SQS（简单队列服务）

| 概念 | 要点 |
|------|------|
| **类型** | 标准 (无限 TPS · 至少一次) · FIFO (300 TPS · 严格一次) |
| **可见性超时** | 消息被消费后对其他消费者不可见的时间 |
| **死信队列 (DLQ)** | 多次消费失败后移入，用于调试 |
| **长轮询** | WaitTimeSeconds=20 减少空轮询成本 |
| **加密** | KMS 静态加密 + HTTPS 传输中 |

### 标准 vs FIFO
| | 标准 | FIFO |
|--|------|------|
| 吞吐量 | 无限 | 300 msg/s (批处理 3000) |
| 顺序 | 不保证 | 严格 FIFO |
| 重复 | 至少一次 | 仅一次 |
| 场景 | 解耦 · 削峰填谷 | 银行交易 · 事件排序 |

## SNS（简单通知服务）

| 概念 | 要点 |
|------|------|
| **模型** | 发布/订阅 (Pub/Sub) |
| **端点** | SQS · Lambda · HTTP(S) · 邮件 · SMS · 移动推送 |
| **消息筛选** | 订阅过滤器策略 (Filter Policy) |
| **FIFO 主题** | 严格排序 + 去重 (配合 FIFO 队列) |

## Step Functions（工作流编排）

| 概念 | 要点 |
|------|------|
| **类型** | 标准 (1 年) · 快速 (5 分钟) |
| **状态** | Task · Choice · Wait · Parallel · Map |
| **集成** | Lambda · Batch · ECS · DynamoDB · SQS · SNS |
| **场景** | 微服务编排 · 数据处理管道 · 人工审批 |

## API Gateway

| 概念 | 要点 |
|------|------|
| **类型** | HTTP · REST · WebSocket |
| **认证** | IAM · Cognito · Lambda Authorizer (Custom) |
| **缓存** | API 缓存减少后端调用 |
| **节流** | 每秒请求限制 + 突发限制 (Token Bucket) |
| **阶段** | Dev/Staging/Prod — 版本管理 |
| **Canary 部署** | 百分比的流量导向新版本 |

---

**SAA 经典架构：** S3 + SQS + Lambda + DynamoDB = 完全无服务器后端
**SAP 经典架构：** Step Functions + SQS DLQ + SNS 告警 = 可靠编排任务
