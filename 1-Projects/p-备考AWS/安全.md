# 🔒 安全

## IAM（身份与访问管理）

| 概念 | 要点 |
|------|------|
| **用户** | 长期凭证（密码 + 访问密钥）|
| **组** | 用户集合，权限继承 |
| **角色** | 临时凭证 (STS) — 跨账户/服务授权 |
| **策略** | JSON 格式 · 白名单/黑名单 · 支持条件键 |
| **策略评估逻辑** | 默认 Deny → 显式 Deny > 显式 Allow |
| **SCP** | 组织级别权限边界 (OU 或账户) |

### IAM 最佳实践
- 最小权限原则
- 使用角色替代长期访问密钥
- 定期轮换密钥
- 使用条件键 (SourceIP, MFA, Tag)
- 不创建 IAM 用户，用 IAM Identity Center (SSO)

### 信任策略 vs 权限策略
- **信任策略** — 谁可以担任此角色 (Principal)
- **权限策略** — 担任角色后可以做什么

## KMS（密钥管理服务）

| 概念 | 要点 |
|------|------|
| **CMK** | 主密钥 (AWS 托管/客户管理/Custom Key Store) |
| **自动轮换** | AWS 托管 CMK 自动轮换 · 客户管理 CMK 可选每年 |
| **信封加密** | KMS 加密 DEK (数据加密密钥) → 用 DEK 加密实际数据 |
| **CloudHSM** | 专用硬件安全模块 (FIPS 140-2 Level 3) |

## WAF（Web 应用防火墙）

| 概念 | 要点 |
|------|------|
| **防护** | SQL 注入 · XSS · 常见 Web 攻击 |
| **关联** | CloudFront · ALB · API Gateway · AppSync |
| **规则** | IP 限制 · 地域限制 · 速率限制 · 规则组 |
| **Web ACL** | 规则集合，默认 Block 后 Allow |

## AWS Shield

| 等级 | 防护 |
|------|------|
| **Shield Standard** | 免费 · L3/L4 DDoS 防护 (SYN/UDP/反射) |
| **Shield Advanced** | ~$3000/月 · L7 防护 · DDoS 响应 · 成本保护 |

## Cognito（身份认证）

| 组件 | 用途 |
|------|------|
| **User Pool** | 用户注册/登录 (用户目录) |
| **Identity Pool** | 临时 AWS 凭证 (授予访问 AWS 服务) |
| **Federation** | Social (Google/FB) · SAML · OIDC |
