# 📊 分析

## Athena（无服务器查询）

| 概念 | 要点 |
|------|------|
| **引擎** | Presto — 标准 SQL |
| **数据源** | S3 中的数据 (CSV/JSON/Parquet/ORC) |
| **计费** | 按扫描数据量收费 |
| **分区** | 分区表可大幅降成本和提升速度 |
| **压缩** | Parquet / ORC 列式存储 + 压缩 = 性能最好 |

## Redshift（数据仓库）

| 概念 | 要点 |
|------|------|
| **架构** | 领导者节点 + 计算节点 |
| **列式存储** | 适合分析查询 (非 OLTP) |
| **Redshift Spectrum** | 直接查询 S3 数据 (无需加载) |
| **备份** | 自动备份到 S3 (1-35 天) |
| **并发** | WLM (工作负载管理) |
| **RA3 节点** | 计算与存储分离，自动扩展 |

> **Athena vs Redshift：** Athena = 无服务器查询 S3 · Redshift = 数据仓库，适合复杂分析和大规模 JOIN

## Kinesis（实时流处理）

| 组件 | 用途 |
|------|------|
| **Data Streams** | 实时数据摄取 (Shard 扩容) |
| **Data Firehose** | 流数据→S3/Redshift/Elasticsearch (近实时) |
| **Data Analytics** | 流上运行 SQL/Java 分析 |
| **Video Streams** | 视频流摄取 |

## EMR（大数据平台）

| 概念 | 要点 |
|------|------|
| **框架** | Spark · Hadoop · Hive · HBase · Presto · Flink |
| **存储** | S3 作为数据湖 + EC2 作为计算 (临时) |
| **模式** | 长期集群 · 临时集群 · 自动伸缩 |
| **Master 节点** | YARN ResourceManager + HDFS NameNode |
| **Core 节点** | YARN NodeManager + HDFS DataNode |
| **Task 节点** | 仅计算 (无存储)，可 Spot 实例 |
