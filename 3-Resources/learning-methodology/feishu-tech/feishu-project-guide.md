# 🏃 飞书：你的执行引擎

> 飞书管的是「做没做」，不管「懂没懂」。它是你的行动中心，不是知识中心。

---

## 一、心法

飞书在四工具里的定位最清晰也最容易被乱用：

- ✅ 用飞书管**项目推进**、**任务打卡**、**日程安排**、**数据追踪**
- ❌ 用飞书记笔记（碎片化、无链接）、用飞书做知识管理（搜索差、无网络）

**一句话：飞书是你每天工作的操作台，Obsidian 是你长期记忆的图书馆。**

---

## 二、飞书四大核心工具

### 1. 多维表格（最重要‼️）

擅长：结构化的数据追踪。比 Excel 灵活，比数据库易用。

**学习管理的多维表格结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 计划哪天做 |
| 模块 | 单选 | 听力/阅读/写作/口语/语法 |
| 任务 | 文本 | 具体做什么 |
| 预估时长 | 数字 | 分钟 |
| 实际时长 | 数字 | 做完了填 |
| 完成 | 复选框 | ✅ |
| 产出 | 文本 | 做完产生了什么（Obsidian 笔记链接、作文等） |
| 笔记链接 | 链接 | Obsidian 对应笔记路径 |

### 2. 日历

- 模考预约 → 飞书日历
- 每日学习时间段 → 飞书日历
- 定期回顾 → 飞书日历（每周日晚 30 分钟）

### 3. 飞书文档

- 存放大型学习资料（PDF 真题、听力音频链接、课程大纲）
- 周报/月报汇总文档（Claude 生成，手动贴入）

### 4. 机器人/提醒

- 每日早 8:00 — 今日学习计划推送
- 每日晚 22:00 — 学习打卡提醒
- 每周日 20:00 — 周回顾提醒

## 三、跟 Claude + Obsidian 的协作

### 信息流

```
早晨                    学习时段                   晚上
  │                        │                        │
  ▼                        ▼                        ▼
飞书推送             Claude 费曼对话           飞书打卡弹窗
查看今日任务          Obsidian 存储成果          填写实际完成情况
                    飞书记录完成状态            对 Claude 说：
                                              "帮我总结今天学了什么"
                                              →
                                              Obsidian 每日笔记
```

### 模考循环

```
飞书日历 → 安排模考
    ↓
模考结束 → 成绩填飞书表格
    ↓
飞书表格数据 → 复制给 Claude："帮我分析我的模考趋势"
    ↓
Claude 生成趋势分析 + 改进建议 → 存 Obsidian 错题本
    ↓
Claude 建议下周重点 → 更新飞书下周日历安排
```

## 五、与其他工具的衔接表

| 信息从哪来 | 操作 | 飞书里怎么体现 |
|-----------|------|---------------|
| Claude 费曼对话结束 | 存入 Obsidian | 飞书该任务标记完成 + 附上笔记链接 |
| Obsidian 学习日志 | daily-review | 飞书填实际时长 |
| 模考成绩 | Claude 分析趋势 | 飞书表格更新成绩 + 附分析结论 |
| Claude 生成周报 | 手动贴入 | 飞书周报文档 |
| 新学习想法 | 丢入 Obsidian Inbox | 飞书任务列表加一条「待整理」 |

## 六、避坑指南

| 坑 | 为什么 | 正确做法 |
|----|--------|----------|
| 飞书管得太细 | 每 15 分钟打一次卡 → 心理负担 | 一天 3-5 个大任务块即可 |
| 把学习笔记写在飞书 | 搜索不了，无法关联 | 笔记全在 Obsidian |
| 飞书多维表格太复杂 | 花 2 小时建表，没时间学习 | 先用简单结构，边用边加字段 |
| 打卡变成形式主义 | 点了完成但没真学 | 勾选前强制自己填「实际产出」 |
