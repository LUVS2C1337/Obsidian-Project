# 📚 如何用四工具读透一本书

> 读完一本书 ≠ 读懂一本书。四工具帮你从「翻过」到「内化」。

---

## 完整流程

### Phase 1: 预读（30 分钟，飞书 + Claude）

**飞书任务：**
```
任务：预读《[书名]》
步骤：
□ 看目录，了解结构
□ 读序言/前言，了解作者意图
□ 翻一遍，标记感觉重要的章节
□ Claude 对话：预读准备
```

**Claude 对话：**
> 「我准备读《[书名]》，作者是 [X]。我想了解：
> 1. 这本书在它所属领域的位置是什么？
> 2. 读之前我需要知道哪些背景知识（不需要剧透）？
> 3. 给我 3 个引导问题，让我带着问题去读。」

**输出：** 在 Obsidian 创建 `3-Resources/[书名] MOC.md`，预设阅读问题。

---

### Phase 2: 逐章精读（每天 50 分钟）

**循环（每章一次）：**

```
Step 1: 读一章（25 分钟）
Step 2: 合上书，打开 Claude（15 分钟）
  "我刚读完《[书名]》第 X 章，核心内容是 [我的复述]。
   请检查我的理解，纠正偏差。"
Step 3: Claude 追问（5 分钟）
  "针对这一章，问我 3 个问题检验理解。"
Step 4: 存入 Obsidian（5 分钟）
  "把这章的关键洞察、我的理解、你的纠正，整理成一篇笔记。
   命名：'第 X 章 - [章节名]'"
Step 5: 在 Obsidian 里手动加 [[链接]]
```

**飞书：** 每完成一章，在多维表格打勾。

---

### Phase 3: 全书消化（周末 60 分钟）

**NotebookLM（如果有 PDF）：**
1. 上传全书 PDF
2. 「全书的核心论点的论证链是什么？」
3. 「哪些章节之间有关联或呼应？」
4. 生成音频概览，做其他事时听一遍

**Claude 综合对话：**
> 「我读完了《[书名]》全书。我一周读了 [X] 天，每天学了 [Y] 章。
> 现在帮我做：
> 1. 梳理全书的论证结构（主要论点 → 分论点 → 证据）
> 2. 指出我 Obsidian 笔记中遗漏或薄弱的理解
> 3. 把书中的知识跟我已有的知识体系关联
> 4. 这本书对我正在做的 [项目/考试] 有什么启发？」

---

### Phase 4: 产出（60 分钟）

不要读完就算了。用其中一个方式输出：

| 方式 | 用哪个工具 | 怎么做 |
|------|-----------|--------|
| 写一篇书评 | Claude 批改 + Obsidian 存储 | 500 字，用自己的话讲这本书 |
| 做一次分享 | 飞书安排 + Claude 打磨 | 15 分钟分享大纲 |
| 画一张知识图 | Obsidian 画布 (Canvas) | 把笔记节点连成网络 |
| 写一篇行动清单 | 飞书记任务 | 书中可以直接用的 3 个行动 |
