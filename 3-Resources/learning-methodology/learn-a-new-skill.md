# 💡 如何用四工具学习一项新技能

> 学技能跟学知识不一样：知识追求懂，技能追求能。四工具的用法也不同。

---

## 知识与技能的关键区别

| | 知识（Know-that） | 技能（Know-how） |
|------|------|------|
| **例子** | 历史、生物学、语法规则 | 编程、弹琴、设计、公开演讲 |
| **评判标准** | 能不能讲清楚 | 能不能做出来 |
| **Claude 的角色** | 费曼陪练、纠错 | 导师、代码 review、反馈教练 |
| **Obsidian 的角色** | 知识网络 | 项目日志 + 错误库 + 原则库 |

---

## 技能学习的三阶段

### 阶段 1: 认知期（我懂了，但做不出）

**目标：** 建立心智模型。

**Claude：**
> 「我想学 [技能]。给我：
> 1. 一个学习路线图，按先后顺序排列
> 2. 新手最容易卡住的 3 个地方 + 怎么破
> 3. 一个最小可行的第一个项目（小到 1-2 天能完成）」

**Obsidian：** 记录学习路线图，创建 `[技能] 学习日志`。

**飞书：** 把路线图拆成周任务，排入日历。

**NotebookLM（如果有教程文档）：** 上传多个教程，交叉对比不同作者的解释。

---

### 阶段 2: 练习期（我能做出来，但很慢/很糙）

这是最长的阶段。关键是**高频反馈**。

**Claude 的用法变了——不再是费曼，而是反馈教练：**

> 编程：
> 「我写了这段代码：[代码]。Review 它，指出：bug、设计问题、更好的写法。」

> 写作：
> 「我写了这段文字：[文字]。从结构、节奏、说服力三个维度批改。」

> 设计：
> 「这是我想达到的效果 [描述/AI生成的效果图]，这是我的尝试 [你的图]，差距在哪里？怎么迭代？」

**Obsidian 的角色也变了——变成项目日志：**

每天结束，在 Obsidian 记录：
```markdown
## [技能] 学习日志 - YYYY-MM-DD

### 今天做了什么
- 

### 遇到了什么问题
- 

### 怎么解决的 / Claude 给了什么反馈
- 

### 明天计划
- 
```

**飞书：** 追踪练习时长。技能的进步几乎完全正比于有效练习时长。

---

### 阶段 3: 熟练期（我能做出来，而且比较快）

**Claude 的用法继续升级——变成高级合作者：**

> 「这段代码/这个设计还有什么 edge case 我没考虑到？」

> 「我现在的水平是什么样的？对应这个领域的能力模型中什么级别？下一级需要什么？」

> 「给我一个更复杂、更接近真实世界的项目来挑战自己。」

**Obsidian 积累「原则库」：**

学了够多之后，开始提炼自己的原则：

```markdown
## [技能] — 我的原则

### 关于架构
1. 先跑通再优化（不要 premature optimization）
2. 接口先于实现
3. ...

### 关于 debug
1. 问题不出在你觉得最可能的地方
2. ...
```

这些原则是你花了几百小时练出来的，是最值钱的东西。

## 具体技能示例

### 学编程

| 工具 | 用法 |
|------|------|
| Claude | 解释概念 + Code review + 帮你 debug + 教你更好的写法 |
| Obsidian | 概念笔记 + 项目日志 + 常见 bug 库 + 原则库 |
| 飞书 | 学习路线追踪 + 每日编码时间打卡 |
| NotebookLM | 读官方文档/技术书籍的 PDF |

### 学外语（你不是刚搭了雅思吗）

| 工具 | 用法 |
|------|------|
| Claude | 口语陪练 + 作文批改 + 解释语法 + 纠音建议 |
| Obsidian | 词汇网络 + 语法知识库 + 错题本 + 语料库 |
| 飞书 | 每日练习打卡 + 模考安排 + 成绩追踪 |
| NotebookLM | 上传真题 PDF 和教材，交叉分析 |

### 学公开演讲

| 工具 | 用法 |
|------|------|
| Claude | 帮你打磨大纲 + 模拟听众提刁钻问题 + 分析你的逐字稿 |
| Obsidian | 演讲素材库、金句库、不同场景的演讲稿模板 |
| 飞书 | 安排练习时间 + 追踪演讲机会（TEDx、公司分享等） |
| NotebookLM | 分析优秀演讲稿的 PDF |

---

## 最重要的原则

> **学技能的时候，Claude 对话时间应该小于你实际练习的时间。**

知识：Claude 对话 ≈ 学习。因为你通过对话在思考。

技能：只有做的过程才算学习。Claude 是教练，不是替身。让它批评你的输出，不要让它替你输出。
