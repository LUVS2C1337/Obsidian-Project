# 🚀 四工具学习系统：操作手册

> 5 分钟读完，终身受用。记住一句话：**飞书管做没做，Claude 管懂没懂，Obsidian 管记住没，NotebookLM 管读透没。**

---

## 一、你需要做什么

| 时间 | 动作 | 工具 | 耗时 |
|------|------|------|------|
| 早晨 | 看飞书今日任务清单 | 飞书 | 2 min |
| 每次学完一个知识点 | **费曼对话**（见下方对话模板） | Claude | 10-15 min |
| 费曼结束后 | 说「把这个知识点归档到 Obsidian」 | Claude → Obsidian | 1 min |
| 存入后 | 在 Obsidian 里手动加 1-2 个 `[[链接]]` | Obsidian | 2 min |
| 任务完成后 | 飞书打勾 + 填产出（学了什么的具体记录） | 飞书 | 1 min |
| 晚上 | 对 Claude 说「总结今天最重要的 3 个收获」 | Claude | 5 min |
| 周日晚上 | 周回顾：Claude 分析本周学习日志 | Claude + Obsidian | 30 min |

**每天总投入：约 30 分钟给「学习系统本身」，换来的是学什么都记得住、讲得出。**

## 二、核心动作详解

### 动作 1：费曼对话（最重要‼️）

**模板（直接复制使用）：**

```
我刚学了 [主题名称]。

让我用自己的话解释一遍：
[你的理解，200-500 字]

请你：
1. 指出我哪里理解正确
2. 指出我哪里理解有偏差，偏差在哪
3. 用最精准的方式纠正我
4. 最后问我 3 个追问，检验我是真懂还是假懂
```

**为什么这是最重要的：** 你以为懂的东西，一开口就露馅。Claude 帮你精准定位「假懂区」。这一句话模板的价值超过你报过的任何补习班。

---

### 动作 2：知识归档

**模板：**

```
把刚才讨论的 [知识点]，整理成 Obsidian 笔记格式。

要求：
- 200-400 字
- 用自己的话写
- 包含 frontmatter（date, tags, aliases）
- 包含「一句话总结」和「为什么重要」
- 末尾给出 2-3 个建议链接的主题

保存到 Obsidian，位置：[如 3-Resources/雅思/写作/]
```

---

### 动作 3：加链接

Claude 存入笔记后，打开 Obsidian，花 2 分钟：

1. 读一遍 Claude 写的笔记
2. 改几个措辞让它真正变成「你的话」
3. 找到 1-2 篇已有笔记，加上 `[[链接]]`

**加链接时问自己三个问题：**

| 问题 | 示例 |
|------|------|
| 这个概念由什么构成？ | 雅思写作 ← [[TR 任务回应]] [[CC 连贯衔接]] |
| 这个概念是哪个更大概念的一部分？ | 虚拟语气 ← [[雅思语法体系]] |
| 这个概念和什么有关系？ | 听力同义替换 ← [[阅读同义替换]] |

---

### 动作 4：周日晚回顾

**模板：**

```
帮我回顾这周 Obsidian 的学习日志和错题记录。

输出：
1. 本周学习时间分布（各模块占比）
2. 重复出现的薄弱点（出现了 2 次以上的错误类型）
3. 本周进步最大的一个模块
4. 下周重点建议（具体到每天做什么）
5. 一份飞书周报格式的总结（我直接复制到飞书）
```

## 三、三种学习场景完整流程

### 场景 A：学一个新的理论知识

| 步骤 | 做什么 | 工具 | 示例 |
|------|--------|------|------|
| 1 | 获得材料 | 书/视频/文章 | 看到一篇讲「认知负荷理论」的文章 |
| 2 | 费曼对话 | Claude | 「让我用自己的话解释认知负荷理论…」 |
| 3 | Claude 纠偏 | Claude | 「你的理解基本对，但你漏了三种负荷的区别」 |
| 4 | 追问检验 | Claude | 「问我 3 个问题检验我是否真懂」 |
| 5 | 归档 | Claude→Obsidian | 「把这个知识点整理成笔记，存到 3-Resources」 |
| 6 | 加链接 | Obsidian | 链接到 `[[学习效率]]` `[[工作记忆]]` |
| 7 | 标记完成 | 飞书 | 打勾 + 产出栏写「Obsidian: 认知负荷理论」 |

### 场景 B：刷题 + 错题分析

| 步骤 | 做什么 | 工具 | 示例 |
|------|--------|------|------|
| 1 | 做题 | 真题/习题集 | 做了一套雅思阅读，错 5 道 |
| 2 | 错题分析 | Claude | 「这 5 道题我错了，帮我分析每道错因」 |
| 3 | Claude 诊断 | Claude | 「3 道是同义替换没识别，2 道是定位偏差」 |
| 4 | 针对性练 | Claude | 「针对同义替换，给我出 5 道练习」 |
| 5 | 归档错题 | Claude→Obsidian | 「把这些错题分析归档到错题本」 |
| 6 | 加链接 | Obsidian | 链接到 `[[阅读同义替换]]` `[[定位技巧]]` |
| 7 | 记录成绩 | 飞书 | 模考表格填分：R=35/40 |

### 场景 C：读一本书/一篇论文

| 步骤 | 做什么 | 工具 | 示例 |
|------|--------|------|------|
| 1 | 预读准备 | Claude | 「我要读《原子习惯》，给我 3 个引导问题」 |
| 2 | 读一章 | 书 | 读了第 1 章「习惯的复利效应」 |
| 3 | 费曼该章 | Claude | 「这一章的核心是…让我用自己的话讲」 |
| 4 | 归档章笔记 | Claude→Obsidian | 「存到 3-Resources/原子习惯/第1章」 |
| 5 | （如有 PDF） | NotebookLM | 上传全书，交叉对比理解 |
| 6 | 全书读完后 | Claude | 「帮我梳理全书论证结构 + 我的薄弱理解」 |
| 7 | 产出书评/行动 | Obsidian + 飞书 | 写 500 字书评存 Obsidian，3 个行动放飞书 |

## 四、飞书多维表格怎么建

打开飞书 → 新建多维表格 → 按以下字段建：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 日期 | 日期 | 计划哪天学 |
| 模块 | 单选 | 听力 / 阅读 / 写作 / 口语 / 词汇 / 语法 |
| 任务内容 | 文本 | 具体做什么 |
| 预估时长(min) | 数字 | 计划多久 |
| 实际时长(min) | 数字 | 实际多久 |
| 完成 | 复选框 | ✅ |
| 产出 | 文本 | 学了什么？（填 Obsidian 笔记名） |
| Obsidian链接 | 链接 | 对应笔记路径 |

**视图建议：**
- 看板视图：按「完成」分组 → 一眼看今天哪些没做
- 日历视图：按「日期」→ 看每周安排密度

## 五、避坑清单

| 坑 | 后果 | 正确做法 |
|----|------|----------|
| 费曼对话跳过 | 以为懂了其实没懂，考试时暴露 | **每天至少一次，这是底线** |
| 直接复制 Claude 输出当笔记 | 你的大脑没有参与加工 | 必须用自己的话改写一遍 |
| 笔记不加链接 | 知识孤岛，后期搜索找不到 | 每篇至少一个 `[[链接]]` |
| 飞书打勾不填产出 | 自我欺骗，数据没有意义 | 强制填「产出」再打勾 |
| 学完不复盘 | 学 10 小时 = 记住 2 小时的内容 | 周日晚 30 分钟回顾 |
| 收藏癖（光存不看） | Obsidian 变成数字坟场 | 只有费曼通过的才值得存 |
| 追求完美分类 | 花 2 小时分类，0 分钟学习 | 先丢 Inbox，每周清理一次 |

## 六、一句话记住这个系统

```
┌────────────────────────────────────────────────┐
│                                                │
│   学一个东西 → 用自己的话讲给 Claude 听          │
│   Claude 挑错 → 纠正理解 → 整理成笔记            │
│   笔记存 Obsidian → 加链接 → 飞书打勾            │
│   周日回顾 → 调整下周计划                        │
│                                                │
│   如果今天只做一件事：费曼对话。                  │
│                                                │
└────────────────────────────────────────────────┘
```

## 七、现在就可以试

打开 Claude，直接说：

> 「我现在想练习一下费曼对话。我最近学到的或读到的一个东西是：[任何你最近接触的内容]。让我用我自己的话解释给你听。」

做完这一次，你就知道这个系统的威力了。
