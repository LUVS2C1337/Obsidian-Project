# 📝 Obsidian：打造第二大脑

> Obsidian 不是笔记软件，是**思考的脚手架**。关键不是存了多少笔记，而是生成了多少连接。

---

## 一、心法

### ❌ 错误用法

- 把 Obsidian 当 Word 用（写了一堆互相独立的长文档）
- 把 Obsidian 当收藏夹用（剪藏从来不回看）
- 追求完美文件夹分类（花更多时间整理而不是思考）

### ✅ 正确用法

- 每篇笔记很短（200-500 字），只讲**一个概念**
- 大量的 `[[双向链接]]` 穿起知识网络
- 不追求完美，**先写再连再改**

---

## 二、PARA + Zettelkasten 混合法

### PARA 框架（你已在用）

```
0-Inbox         ← 入口：想法、Claude 对话、临时笔记
1-Projects      ← 有 deadline 的事（雅思备考、学 Rust）
2-Areas         ← 持续维护无 deadline（英语、健康、财务）
3-Resources     ← 主题知识库（学习方法、技术、雅思）
4-Archives      ← 已完成/搁置
```

### Zettelkasten 原子化原则

每篇笔记做到：
- **一笔记一概念**：不在一篇笔记里塞多个不相关的内容
- **用自己的话写**：绝不直接复制 Claude 的输出
- **大量链接**：至少链向 1-2 篇相关笔记

### 实操示例

❌ 写了一篇《人工智能综述》，5000 字，涵盖历史、技术、应用、伦理
→ 太重，无法作为节点参与网络

✅ 拆成：
- `神经网络基础`
- `Transformer 架构`
- `GPT 的训练方式 — 预训练 + 微调 + RLHF`
- `AI 对齐问题 — 让 AI 的价值观符合人类`
每篇 300-500 字，互相链接。

## 三、笔记的原子结构

### Claude 友好模板

每篇笔记都应该包含 Claude 能直接理解的元数据：

```markdown
---
date: "YYYY-MM-DD"
tags: [topic, subtopic]
aliases: [其他叫法1, 其他叫法2]
status: seedling | budding | evergreen
---

# 笔记标题

## 一句话

<!-- 用一句话说清楚这个笔记的核心 -->

## 展开

<!-- 核心概念展开，200-500 字 -->

## 为什么重要

<!-- 这个知识点跟什么有关联，为什么要理解它 -->

## 相关

- [[相关笔记1]]
- [[相关笔记2]]
```

### 状态标记

- 🌱 **seedling** — 新创建的，内容还不完整
- 🌿 **budding** — 已有基本框架，在生长中
- 🌲 **evergreen** — 已经比较完善的长期笔记

## 四、链接策略

### 链接的类型

```
[[某个笔记]]                    ← 概念关联
[[某个笔记|显示的文字不一样]]    ← 语境化链接
[[某个笔记#具体段落]]           ← 段落级链接
```

### 链接什么？

在写完每篇笔记后，问自己三个问题：
1. 这个概念由什么构成？→ 链到子概念
2. 这个概念是哪个更大概念的一部分？→ 链到父概念
3. 这个概念和什么有关系？→ 链到相关概念

**如果你说不出一篇笔记可以链到哪里，说明你对这个知识的理解还没有网络化。**

---

## 五、MOC（内容地图）

MOC = Map of Content，是一个笔记的「目录导航」。

你已经有了：
- 写作 MOC、口语 MOC、听力 MOC 等（雅思）
- 项目索引、资源索引等（PARA）

**创建 MOC 的时机：** 当某个主题下积累了 5+ 篇笔记，感觉需要导航时。

**MOC 写法：**
```markdown
# 某个主题 MOC

## 核心概念
- [[概念A]]
- [[概念B]]

## 进阶
- [[进阶C]]

## 待探索
- [ ] 还没有笔记但想了解的方向
```

## 六、与 Claude 的高效配合

### 写入流

| 场景 | 怎么做 |
|------|--------|
| 学完一个概念 | 跟 Claude 费曼 → 确认理解正确 → "帮我组织成 Obsidian 笔记格式" → "保存到 Obsidian" |
| 问题驱动 | "帮我查一下 Obsidian 里我关于 X 的笔记" → Claude 读取旧笔记 |
| 知识连接 | "我学了 Y，跟我已有的 X 笔记有什么关联？" → Claude 读取 X，给出连接建议 |
| 定期复习 | "回顾我这周/这个月的学习日志，找出我应该重点复习的薄弱点" |

### 不要做的

- ❌ 让 Claude 直接写 5000 字长文塞进 Obsidian
- ❌ 不检查直接存（Claude 可能有细微偏差）
- ❌ 存了不加链接（等于死数据）

## 七、三种笔记，不同策略

| 类型 | 来源 | 频率 | 处理方式 |
|------|------|------|----------|
| **闪念** | 突然的想法 | 随时 | 丢进 Inbox，每天清空 |
| **文献笔记** | 书/文章/视频 | 阅读时 | 用自己的话写 3-5 条关键洞察 |
| **永久笔记** | 思考消化后 | 每周 2-3 篇 | 原子化 + 大量链接 + Evergreen 标记 |
