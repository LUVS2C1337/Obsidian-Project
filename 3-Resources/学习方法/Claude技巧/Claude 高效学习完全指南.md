# 🤖 Claude：你的 24 小时私教

> Claude 不是搜索引擎，是**思维陪练**。关键不是问它问题，而是让它帮你思考。

---

## 一、心法：你跟 Claude 的关系

### ❌ 错误用法

```
你：量子力学是什么？
Claude：...（一段解释）
你：好的谢谢。← 什么都没学到
```

这是把 Claude 当 Google。你得到的是**信息**，不是**理解**。

### ✅ 正确用法

```
你：我刚看了一篇关于量子纠缠的文章，我理解是两个粒子不管
    距离多远都能瞬间影响对方。我用我自己的话解释一遍，
    你帮我看看哪里理解错了。

Claude：基本框架对，但「瞬间影响」这个表述需要修正。
      实际上信息不能超光速传输。不如你用这个比喻试试：
      假设你有两张牌...

你：那我重新理解一下...（再复述）

Claude：这次对了。你掌握得不错。接下来要不要深入一下
      为什么爱因斯坦当年不接受这个理论？
```

**核心区别：你把 Claude 当成 sparring partner（陪练伙伴），不是答案机器。**

## 二、四大核心学习模式

### 模式 1：费曼练习（最重要‼️）

学完任何新东西后，立刻对 Claude 说：

> 「我刚学了 [主题]，让我用自己的话解释一遍。你帮我检查：哪几理解正确，哪几理解错误或不够准确，然后纠正我。」

**为什么有效：** 教是最好的学。你以为懂的，一开口就露馅。Claude 帮你精准定位「假懂区」。

**进阶版：** 让 Claude 追问你

> 「关于 [主题]，请连续问我 5 个问题来检验我是不是真的理解透彻。从简单开始，逐步深入。如果某一层我答不上来了，就在那一层展开讲解。」

### 模式 2：苏格拉底对话

> 「我想理解 [概念]，但不要直接告诉我答案。用苏格拉底问答法引导我，让我自己推理出来。」

**适用：** 概念性知识、批判性思维、伦理判断

**示例效果：**
```
Claude：你觉得，一个人做一件事纯粹出于习惯，和出于理性选择，
       有没有本质区别？
你：我觉得...
Claude：那如果一个习惯最初是由理性选择形成的呢？现在算习惯还是选择？
你：（开始深度思考）
```

### 模式 3：对比辨析

学习中最容易混淆的概念，让 Claude 帮你拉开：

> 「[概念 A] 和 [概念 B] 经常被搞混。帮我做三个层次的对比：
> 1. 一句话说出核心区别
> 2. 用生活化例子各举一个
> 3. 给我一个判断题来检验」

**示例：** 影响 vs 效果、相关性 vs 因果、准确率 vs 召回率……

### 模式 4：场景迁移

> 「我已经理解了 [概念] 的基础。现在给我 3 个让我意想不到的、跨领域的应用场景，看看我能不能应对。」

## 三、按学习阶段的提示词模板

### 📥 预习阶段

```
我马上要学 [主题]。帮我：
1. 先给我一个 200 字的「外行友好版」概述
2. 列出 3 个核心问题，让我带着问题去学
3. 提醒我容易踩的 2 个理解误区
```

### 📖 学习过程中

```
我刚才读到 [具体内容]，卡住了。
我理解的意思是 [我的理解]。
但我觉得好像不太对，因为 [矛盾点]。
帮我厘清。
```

### ✍️ 巩固阶段

```
我现在学完了 [主题]。请：
1. 给我 3 道变式题检验我
2. 让我用一句话总结核心思想，你评估我的总结
3. 帮我把这个知识点和 [之前学过的内容] 建立联系
```

### 🧪 实战阶段

```
我已经掌握了 [知识]，现在想用它来解决这个实际问题：
[具体问题描述]

我先说说我的思路：[你的思路]
你帮我评估思路，然后给一个优化方案，但不要直接给答案。
```

## 四、Claude 的独门绝活

### 1. 个性化类比生成

> 「用我熟悉的领域来打比方：[概念]。我熟悉 [你的领域，如篮球/烹饪/编程/游戏]。」

### 2. 错误案例教学

> 「关于 [概念]，给我 5 个看起来正确但其实是错误的说法。让我逐个判断，然后你告诉我为什么。」

**研究证明：** 让人辨别错误比直接教正确更有效。

### 3. 分层解释

> 「用四个层次的复杂度解释 [概念]：
> L1：给小学生
> L2：给高中生
> L3：给大学生
> L4：给专家
> 在每层之后停下来，让我复述。」

### 4. 反向教学

> 「假装你对 [主题] 一窍不通，让我来教你。你扮演一个聪明但爱问为什么的学生，不断追问直到我讲不透为止。」

### 5. 思维过程复现

> 「在解决 [问题] 时，不要直接给我最终答案。请展示你完整的思考过程：每一步的假设、尝试、淘汰的方案、最终选的路径和理由。」

## 五、避坑指南

| 坑 | 为什么 | 正确做法 |
|----|--------|----------|
| 让 Claude 帮你写作业 | 你跳过了思考 | 让它批改你的作业 |
| 复制 Claude 的回答当笔记 | 你没有转化 | 理解后用**自己的话**重写 |
| 问太宽泛的问题 | 答案也宽泛无用 | 加入具体语境和你已有的理解 |
| 不追问 | 停在浅层理解 | 追问「为什么」「如果呢」「你怎么知道」 |
| Claude 说「你对」就信 | 它可能遗漏了细微偏差 | 要求它「找出我理解中最不准确的一个点」 |

## 六、跟 Obsidian 联动的最佳实践

### 学习闭环流

```
1. 你学了一个新东西
2. 跟 Claude 费曼对话（15 分钟）
3. 对 Claude 说：「把刚才讨论的核心要点、
   我踩的坑、纠正后的理解，组织成一份结构化的笔记」
4. Claude 输出格式化笔记
5. 你说：「保存到 Obsidian，放在 [位置]」
6. 在 Obsidian 里手动加上 [[相关笔记的链接]]
```

### 每日学习收尾

> 「今天我学了 [A, B, C]。请回顾我们今天的对话，帮我：
> 1. 梳理 3 个最重要的 insight
> 2. 标注我哪个概念还很薄弱
> 3. 整理成 Obsidian 格式的每日笔记」
